## 原理
断点功能一般是通过特定的内核信号实现的，gdb捕获该信号，定位目标程序停止的地址来判断断点是否成功触发。
首先gdb fork()出来一个子进程，该子启动目标程序(通过ptrace() 和 exec())，
父进程捕获该子进程的所有的信号(通过ptrace() 和 wait())，当子进程收到信号时，子进程就会被挂起，直到父进程通知其继续运行(通过ptrace())

## 退出gdb
1 输入quit:

    $ gdb -q
    (gdb) quit

2 输入Ctrl-d:

    $ gdb -q
    (gdb) quit

## 为gdb进行编译
